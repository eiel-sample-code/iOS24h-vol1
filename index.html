インストール

CocoaPodsの紹介記事などをみるとCocoaRuby必要と書かれていることがありますが、 現在はLionにインストール済みのRubyでも利用できます。

必須条件

* rubgems 1.3.6以上 * OS X 10.7以上 * Command Line Tools for Xcodeのインストール

Command Line Tools For Xcode

preferencesよりインストール可能

準備ができたら
インストール

$ gem install cocoapods
$ pod setup

基本的な使い方

Podfileを作成

platform :ios
dependency 'JSONKit'
dependency 'Reachability'
 

$ pod install プロジェクト名.xcodeproj

2度目以降は installだけでOK 

$ pod install

生成されるファイル

* Podfile.lock * Pods * プロジェクト名.xcworkspace

Podfile.lock

使用しているバージョンが記録されています

Pods

ライブラリとライブラリを管理するプロジェクトなどが配置されます。削除してもpods installで再度インストールできます。

プロジェクト名.xcworkspace

プロジェクトを開く際に、xcodeprojの変わりに使用します。xcodeprojではライブラリをみつけることができないため、ビルドできません。

補足

CocoaPodsの開発版では 

$ pod install
のみで良いようです。

試してみる

DCIntrospect

シミュレータ上でUIViewの位置を確認したり、変更したりできます。

https://github.com/domesticcatsoftware/DCIntrospect

Podfile

platform :ios
dependency "DCIntrospect"

Color-Picker-for-iOS

カラーピッカーです。

https://github.com/hayashi311/Color-Picker-for-iOS

Podfile

platform :ios
dependency "DCIntrospect"
dependency "Color-Picker-for-iOS"

どんなライブラリがあるの？

ライブラリ一覧

$ pod list https://github.com/CocoaPods/Specs

検索

$ pod search キーワード

自作ライブラリもPodで

podspecの作成

podspecを作成しておけば、リポジトリを直接指定することで、 一覧にないライブラリの利用もできます。

Podspecの雛形を作成

$ pod spec create 名前

名前.podspec が生成されます。 必要に応じて中身を修正します。

試しにつくってみました https://github.com/eiel/ALPageView

Podfile

platform :ios
dependency "DCIntrospect"
dependency "Color-Picker-for-iOS"
dependency "ALPageView", :git => 'https://github.com/eiel/ALPageViewDemo.git'

もしおかしくなったら

Podsディレクトリを削除して

$ pod install
